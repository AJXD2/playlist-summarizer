# Contributing to Playlist Summarizer

Thank you for your interest in contributing to Playlist Summarizer!

## Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/yourusername/playlist-summarizer.git`
3. Create a new branch: `git checkout -b feature/your-feature-name`
4. Make your changes
5. Test your changes
6. Commit your changes: `git commit -m "Add your meaningful commit message"`
7. Push to your fork: `git push origin feature/your-feature-name`
8. Open a Pull Request

## Development Setup

1. Install dependencies:
```bash
uv sync
```

2. Run the application:
```bash
uv run playlist-summarizer
```

## Code Style

- Follow PEP 8 style guidelines
- Use type hints where appropriate
- Keep functions focused and single-purpose
- Add docstrings for public functions and classes

## Testing

Before submitting a PR, please test your changes to ensure:
- The application runs without errors
- New features work as expected
- Existing functionality is not broken

## Pull Request Guidelines

- Provide a clear description of your changes
- Reference any related issues
- Ensure your code follows the project's style
- Test your changes thoroughly

## Questions?

Feel free to open an issue for any questions or discussions about contributions.
